在大型语言模型或 llm 的开发中，大致有两种类型的 llm，我将其称为基础 llm 和指令调优 llm。

因此，base LLM 已经被训练成基于文本训练数据来预测下一个单词，通常是在来自互联网和其他来源的大量数据上进行训练，以找出下一个最可能跟随的单词是什么。

所以，举个例子，如果你提示我们从前有一只独角兽，它可能会完成这个，也就是说它可能会预测接下来的几个词是住在一个神奇的森林里的所有独角兽的朋友。

但如果你要提示我们法国的首都是什么，那么基于互联网上可能有什么文章，基础法学硕士很可能会完成法国最大的城市是什么，法国的人口是多少等等，因为互联网上的文章很可能是关于法国这个国家的小问题列表。

相比之下，一个指令型 LLM，这是 LLM 研究和实践的很多动力所在，一个指令型 LLM 被训练成遵循指令。

所以，如果你问它法国的首都是什么，它很可能会说，法国的首都是巴黎。

因此，指令调整 LLM 的典型训练方式是，你从一个基本的 LLM 开始，这个 LLM 是在大量的文本数据上训练的，然后进一步训练它，进一步微调它的输入和输出，这些输入和输出是指令，并尝试遵循这些指令，然后通常使用一种叫做 RLHF 的技术进行进一步改进，从人类反馈中强化学习，使系统能够更好地提供帮助并遵循指令。

因为指令调优 LLM 已经被训练成有用、诚实和无害的 LLM，例如，与基础 LLM 相比，它们不太可能输出有问题的文本，比如有毒的输出，因此许多实际使用场景已经转向指令调优 LLM。

你在互联网上找到的一些最佳实践可能更适合基础 LLM，但对于今天的大多数实际应用，我们建议大多数人转而关注指令调整的 LLM，这更容易使用，而且由于 OpenAI 和其他 LLM 公司的工作变得更安全，更一致。

因此，本课程将重点关注指令调优 llm 的最佳实践，这是我们建议您在大多数应用程序中使用的。
