# 扩大

<Player title="prompt_eng_07_expanding_v3" />

扩展任务是取较短的文本，例如一组说明或主题列表，并让大型语言模型生成较长的文本，例如关于某个主题的电子邮件或文章。

这有一些很好的用途，比如如果你使用一个大的语言模型作为头脑风暴的伙伴。

但我也想承认，这有一些有问题的用例，比如如果有人要使用它，他们会产生大量的垃圾邮件。

因此，当您使用大型语言模型的这些功能时，请仅以负责任的方式使用它，并且以有助于人们的方式使用它。

在这个视频中，我们将通过一个例子来说明如何使用语言模型根据一些信息生成个性化的电子邮件。

这封邮件自称来自一个人工智能机器人，正如安德鲁提到的，这是非常重要的。

我们还将使用模型的另一个输入参数，叫做`温度`，这可以让你改变模型响应的探索程度和多样性。

接下来，我们将使用语言模型的一个名为`temperature`的参数，它将允许我们改变模型响应的种类。

所以你可以把温度看作是探索的程度或者模型的随机性。

所以对于这个特定的短语，`我最喜欢的食物是`，模型预测的下一个最可能的单词是`披萨`，而它建议的下一个最可能的单词是`寿司`和`玉米饼`。

因此，在温度为 0 时，模型总是会选择最有可能出现的下一个单词，在这种情况下是`pizza`，在更高的温度下，它也会选择一个不太可能出现的单词，甚至在更高的温度下，它甚至可能选择`tacos`，它只有 5%的可能性被选中。

你可以想象，随着模型继续这个最后的反应，我最喜欢的食物是披萨，它继续产生更多的单词，这个反应会和第一个反应有所不同，第一个反应是我最喜欢的食物是玉米饼。

因此，随着这种模式的继续，这两种反应将变得越来越不同。

通常，在构建需要可预测响应的应用程序时，我建议使用温度零。

在所有这些视频中，我们一直使用零度，我认为如果你想建立一个可靠且可预测的系统，你应该使用这个。

如果你想用一种更有创意的方式来使用这个模型，你可能想要更多种不同的输出，你可能想要使用更高的温度。
